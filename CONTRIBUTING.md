# Contributing to chrome-sync-engine

Thanks for your interest in contributing. This document covers the basics.

GETTING STARTED

1. Fork the repo on GitHub
2. Clone your fork locally
3. Run `npm install` to set up dependencies
4. Create a new branch for your work

DEVELOPMENT

Build the project with:

```bash
npm run build
```

The source lives in `src/` and compiles to `dist/` via TypeScript.

SUBMITTING CHANGES

1. Keep commits focused and descriptive
2. Make sure the project builds cleanly before opening a PR
3. Open a pull request against the `main` branch
4. Describe what your change does and why

REPORTING BUGS

Use the GitHub issue tracker. Include steps to reproduce, expected behavior, and actual behavior. Browser version and extension manifest details help a lot.

FEATURE REQUESTS

Open an issue describing the use case and why it would be valuable. Discussion before implementation saves everyone time.

CODE STYLE

- TypeScript strict mode is enabled
- Keep things simple and readable
- Follow existing patterns in the codebase
